## 四、发现的缺陷（按严重程度）

### 🔴 P1 — 重复用户记录（数据一致性）

**现象：** Coze `users` 表中同一 `openid`（如 `u_1783477493993_7rxc0d`）存在 **2 条记录**。

**原因：** 签到时 `queryUser` 未命中则 `insertUser`，并发或重复点击会多次插入；`queryUser` 只取 `items[0]`，统计可能不准。

**影响：** 个人中心统计数据错误；更新可能只改到其中一条。

**建议：** 插入前强制查询；或改为「有则 update、无则 insert」并加防重复点击锁。

---

### 🟠 P2 — 运势记录未同步云端

**现象：** `fortune_records` 表仅测试数据 1 条，真实用户抽签未落库。

**原因（高概率）：**
1. 微信未配置 **request合法域名**（`api.coze.cn` / `api.minimaxi.com`）
2. 开发者工具未勾选「不校验合法域名」
3. Coze/Minimax 请求失败后走 `catch`，仅用本地缓存 + mock

**影响：** 历史、欧皇榜、个人统计（除签到外）为空或不准。

**建议：** 配置域名后重测；`generateAndSave` 失败时给用户明确 Toast。

---

### 🟠 P2 — Minimax 返回非纯 JSON

**现象：** API 返回英文解释 + Markdown 代码块，而非直接 JSON。

**缓解：** `parseFortuneJson` 已支持提取代码块；失败时 `mockFortune()` 降级。

**影响：** 可能抽到 mock 数据而非真实 AI 内容。

**建议：** 加强 system prompt；或换更听话的模型参数。

---

### 🟡 P3 — 每日限制可被绕过（边缘）

**现象：** `checkTodayOfficial` 在 Coze 查询失败时返回 `null`，仅依赖本地缓存防重。

**影响：** 清缓存后可能同日再次正式抽签。

**建议：** Coze 失败时仍严格以本地 `cacheTodayFortune` 为准（部分已实现）。

---

### 🟡 P3 — 分享海报未提供「保存相册」

**现象：** 仅 `wx.previewImage` 预览，无 `saveImageToPhotosAlbum`。

**影响：** 分享路径多一步，体验略差。

## 六、建议人工复测步骤（15 分钟）

1. 开发者工具勾选「不校验合法域名」→ 重新编译  
2. **首页** 点击玻璃球 → 等待 2s → 确认进入结果页且动画正常  
3. **结果页** 检查分数滚动、SSR 特效（可多抽几次碰 SSR）  
4. **历史** 页应出现刚才的记录  
5. **欧皇榜** 应出现自己的昵称和分数  
6. **我的** → 签到 → 提示成功，连续天数 +1  
7. 再次签到 → 应提示「今日已签到」  
8. 首页「娱乐重抽」→ 确认可进入新结果但不影响正式签状态  
9. 「生成分享海报」→ 确认能出图预览
